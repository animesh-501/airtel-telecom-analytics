# Airtel Telecom Analytics Pipeline

> End-to-end analytics pipeline for telecom customer data — synthetic generation, cleaning, EDA, churn prediction & revenue forecasting.  
> **Animesh Choubey** | [linkedin.com/in/choubey-animesh](https://linkedin.com/in/choubey-animesh)

## Overview

This project simulates a production-grade analytics workflow for an Indian telecom operator (modeled on Airtel). It covers the full data lifecycle from raw synthetic generation through ML-powered churn prediction and 6-month revenue forecasting.

**Domains:** Customer Churn · Revenue/Billing · Support Operations · Fraud-Adjacent Risk Scoring

---

## Pipeline

```
01_data_generation.ipynb   →   airtel_raw_data/
02_data_cleaning.ipynb     →   airtel_clean_data/  +  airtel_telecom.db
03_eda.ipynb               →   airtel_eda_outputs/
04_forecasting.ipynb       →   airtel_forecast_outputs/
```

Each notebook is self-contained and can be run independently (after Phase 1 outputs exist).

## Key Engineering Decisions

- **`config.py`** — all constants (dataset sizes, noise rates, Prophet params, risk weights) live in one file. No magic numbers scattered across notebooks.
- **`utils.py`** — `force_numeric()`, `snapshot()`, `save_fig()`, `compute_risk_score()` are defined once and imported everywhere. No copy-paste.
- **Billing lookup fix** — original code did a pandas row filter inside a 5,000-iteration loop (O(n²)). Replaced with a dict lookup (O(1)).
- **Prophet tuning** — `changepoint_prior_scale` reduced from `0.3` → `0.05` to prevent overfitting on the short (~12 month) training series.
- **YData Profiling** — `minimal=True` by default (much faster). Set `YDATA_MINIMAL=False` in `config.py` for full reports.
- **Risk weights documented** — the `RISK_WEIGHTS` dict in `utils.py` explains what each factor measures and why it's weighted that way.
- **Reproducibility** — all random seeds flow from `SEED = 42` in `config.py`. `inject_noise()` uses a local `RandomState` so it doesn't pollute the global seed.
